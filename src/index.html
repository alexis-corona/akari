2. Programación Orientada a Objetos (POO)

Herencia: Reutilización de código con extends.
Polimorfismo: Sobrecarga y sobreescritura de métodos.

6. Arquitectura de Software

Principios SOLID:
Las 5 reglas de oro para escribir código mantenible y limpio.

Arquitectura en Capas (Layered Architecture):
Separar la aplicación en Controlador/Vista, Lógica de Negocio (Service) y Acceso a Datos (Repository/DAO). Es la base de frameworks como Spring Boot.

Inyección de Dependencias:
Qué es, por qué evita el acoplamiento fuerte y cómo se relaciona con la POO avanzada.

Introducción a Arquitecturas Modernas:
