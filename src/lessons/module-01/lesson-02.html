Variables, Constantes y Tipos de Datos

Explorando las variables, constantes y los tipos de datos fundamentales en Java.

1. Variables

En programación, una variable es un espacio reservado en la memoria RAM de la computadora diseñado para almacenar un valor que puede cambiar durante la ejecución del programa.

Para entenderlo mejor, piensa en la memoria RAM como una fila infinita de casilleros numerados (direcciones de memoria). En lugar de que tú tengas que recordar el número exacto del casillero (por ejemplo, 0x7ffd5), Java te permite asignarle un nombre simbólico (un identificador).

¿Cómo funciona el ciclo de vida de una variable?

  • Declaración: Es cuando le decimos a Java: "Reserva un espacio en memoria para un dato de este tipo y llámalo con este nombre". Ejemplo: int edad;
  • Inicialización: Es el momento en que guardamos el primer valor dentro de ese espacio. Ejemplo: edad = 25;
  • Uso: Es cuando el programa lee el contenido de esa "caja" para realizar cálculos o mostrar resultados.
  • Actualización: Como su nombre lo indica, el valor dentro de la caja puede cambiar a lo largo del tiempo, sobrescribiendo el valor anterior.

💡 Concepto clave: La declaración es obligatoria en Java. A diferencia de lenguajes como Python o JavaScript, Java es "estricto": si no declaras qué tipo de datos contendrá tu variable, el compilador rechazará tu código antes de que siquiera intentes ejecutarlo. Esto previene errores de "tipo" que son muy comunes en otros lenguajes.

2. Constantes: Valores que no cambian

A veces, necesitamos definir valores que deben permanecer inalterables durante toda la ejecución de nuestro programa, como el valor de PI o el número máximo de intentos en un inicio de sesión. Para esto usamos el modificador final. 


        // Una variable normal puede cambiar
        int contador = 10;
        contador = 11; // Permitido

        // Una constante no puede cambiar
        final double PI = 3.14159;
        // PI = 3.14; // ¡Error! El compilador no permitirá este cambio

💡 Convención de nombres: Por convención en Java, las constantes se escriben siempre en MAYÚSCULAS_SOSTENIDAS, separando las palabras con un guion bajo (estilo SCREAMING_SNAKE_CASE). Esto permite identificarlas inmediatamente dentro de un bloque de código largo.

3. El sistema de tipos en Java

A diferencia de lenguajes de tipado dinámico, Java es un lenguaje de tipado estático y fuerte. Esto significa que cada variable y constante debe tener un tipo declarado y este no puede cambiar durante la ejecución. Esto garantiza la integridad de los datos en memoria, ya que Java conoce exactamente cuánto espacio ocupa cada variable antes de que el programa se ejecute.

4. Tipos Primitivos

Java utiliza 8 tipos primitivos que almacenan valores directamente en la pila (stack). Son los bloques constructores más eficientes del lenguaje. A continuación, vemos cómo declarar y asignar valores a cada uno en un contexto real:

  • Números enteros: byte, short, int (el estándar), y long (para valores masivos).
  • Números decimales: float y double (el estándar de precisión).
  • Lógica: boolean (solo puede ser true o false).
  • Caracteres: char (almacena un único carácter Unicode). 

        // Números enteros
        byte diaSemana = 1; 
        short anioNacimiento = 1995; 
        int idUsuario = 102450; 
        long seguidores = 1500000L; // La 'L' indica que es un long
            
        // Números decimales
        float precioProducto = 9.99f; // La 'f' indica que es float
        double balanceCuenta = 2500.50; 
            
        // Lógica
        boolean estaActivo = true; 

        // Caracteres
        char inicial = 'A';

💡 ¿Por qué tantos tipos? La respuesta es eficiencia. Usar un byte ahorra memoria frente a un int. Aunque hoy tenemos mucha RAM, en sistemas de alto rendimiento o embebidos, elegir el tipo correcto es vital.

5. Tipos Objetos (Wrapper Classes)

En Java, los tipos primitivos no son objetos y, por tanto, no tienen métodos. Aquí es donde entran las Clases Wrapper. Cada primitivo tiene su "envoltura" como objeto, lo que nos permite usar funciones poderosas sobre ellos (como convertir a String, comparar con null o realizar conversiones de base).

Mientras que el primitivo es solo un valor crudo en memoria, el objeto es una entidad que nos ofrece herramientas de manipulación:

  • Números enteros: Byte, Short, Integer (el estándar), y Long (para valores masivos).
  • Números decimales: Float y Double (el estándar de precisión).
  • Lógica: Boolean (solo puede ser true, false o null).
  • Caracteres: Character (ofrece métodos para validar letras, números, etc.).
  • Texto: String (no tiene primitivo, es una clase fundamental para cadenas). 

        // Números enteros
        Byte b = Byte.valueOf("127");
        Short s = Short.valueOf("32000");
        Integer i = Integer.parseInt("102450");
        Long l = Long.valueOf(1500000L);
        
        // Números decimales
        Float f = Float.valueOf(9.99f);
        Double d = Double.valueOf(2500.50);
        
        // Lógica
        Boolean bool = Boolean.TRUE; // Constante útil
        
        // Caracteres y Texto
        Character c = Character.valueOf('A');
        String nombre = String.valueOf("Akari");
        
        // Ejemplos de uso de métodos de los Wrappers:
        System.out.println(Character.isDigit(c));     // false
        System.out.println(Integer.toHexString(i));   // "19032"
        System.out.println(Double.max(d, 5000.0));    // 5000.0
        System.out.println(nombre.toUpperCase());     // "AKARI"

💡 Nota: El proceso donde Java convierte automáticamente un primitivo a su objeto se llama Autoboxing, y el proceso inverso (objeto a primitivo) se llama Unboxing. Es una herramienta poderosa, pero ten cuidado: si intentas hacer unboxing de un objeto que es null, tu programa lanzará un error NullPointerException.
