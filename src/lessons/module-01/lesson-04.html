Entrada y Salida de Datos

Dominando la entrada, salida y persistencia básica de datos en Java.

Hasta ahora, nuestro código ha operado de forma aislada, mostrando información únicamente en la consola. Sin embargo, un software profesional debe ser capaz de interactuar con el exterior. En esta lección, aprenderás a gestionar el flujo de datos en Java: desde la captura interactiva de información vía teclado con Scanner, hasta técnicas avanzadas de salida formateada y la persistencia de datos mediante la lectura eficiente de archivos externos con BufferedReader.

1. La Clase Scanner

Scanner es una herramienta versátil que desglosa la entrada de datos en "tokens" (pedazos) simples. Para utilizarla, primero debemos importarla desde java.util y crear una instancia vinculada a System.in. 


        import java.util.Scanner;

        public class EntradaEjemplo {
            public static void main(String[] args) {
                // Creamos el objeto Scanner
                Scanner scanner = new Scanner(System.in);

                System.out.print("¿Cómo te llamas? ");
                String nombre = scanner.nextLine(); // Lee una línea completa

                System.out.print("Ingresa tu edad: ");
                int edad = scanner.nextInt(); // Lee un número entero

                System.out.println("Hola " + nombre + ", tienes " + edad + " años.");
                
                // Siempre es buena práctica cerrar el scanner
                scanner.close();
            }
        }

2. Métodos de lectura comunes

Dependiendo del tipo de dato que necesites, Scanner ofrece métodos específicos. Es fundamental elegir el correcto para evitar errores de tipo:

  • nextLine(): Lee una línea completa de texto (incluyendo espacios).
  • next(): Lee solo una palabra (se detiene al encontrar un espacio).
  • nextInt(): Lee el siguiente valor como entero.
  • nextDouble(): Lee el siguiente valor como decimal.
  • nextBoolean(): Lee un valor booleano (true/false). 

        Scanner scanner = new Scanner(System.in);

        System.out.println("--- Registro de Datos ---");

        // Lectura de nombre completo (usa toda la línea)
        System.out.print("Escribe tu nombre: ");
        String nombre = scanner.nextLine(); 

        // next() solo leerá la primera palabra (el primer color si escribe más de uno)
        System.out.print("Escribe tu color favorito: ");
        String color = scanner.next(); 

        // Lectura de valores numéricos
        System.out.print("Escribe tu edad: ");
        int edad = scanner.nextInt();

        System.out.print("Escribe tu promedio de calificaciones: ");
        double promedio = scanner.nextDouble();

        // Lectura de valor booleano
        System.out.print("¿Tienes mascota? (true/false): ");
        boolean tieneMascota = scanner.nextBoolean();

        // Resumen
        System.out.println("\n--- Datos recibidos ---");
        System.out.println("Nombre: " + nombre);
        System.out.println("Color: " + color + " (Solo se guardó la primera palabra)");
        System.out.println("Edad: " + edad);
        System.out.println("Promedio: " + promedio);
        System.out.println("¿Tiene mascota?: " + tieneMascota);

        scanner.close();

💡 Nota: La lectura de datos es un punto de entrada para errores (como que el usuario escriba letras cuando esperabas números). A medida que avances, aprenderás a manejar estas situaciones con bloques try-catch para validar que la entrada del usuario sea correcta antes de procesarla.

3. Consideración importante: El "Salto de línea"

Un error muy común ocurre al combinar nextInt() o nextDouble() con nextLine(). Estos métodos numéricos no consumen el carácter de salto de línea (el "Enter" que presionas), por lo que el siguiente nextLine() puede leer una cadena vacía restante en lugar de esperar la entrada del usuario. La solución rápida es consumir ese salto restante con un nextLine() adicional: 


        Scanner scanner = new Scanner(System.in);

        System.out.print("Ingresa tu edad: ");
        int edad = scanner.nextInt(); // El usuario escribe 25 + [Enter]
        
        // El '25' se guarda en edad, pero el [Enter] sigue en el buffer
        
        System.out.print("Ingresa tu nombre: ");
        String nombre = scanner.nextLine(); // ¡Error! Lee el [Enter] pendiente y se salta la entrada
        
        
        // Solución 
        
        System.out.print("Ingresa tu edad: ");
        int edad = scanner.nextInt();
        scanner.nextLine(); // Consumimos el salto de línea pendiente
        
        System.out.print("Ingresa tu nombre: ");
        String nombre = scanner.nextLine(); // Ahora sí funciona correctamente

4. Impresión en consola: print vs println

Antes de usar formatos complejos, es vital recordar la diferencia fundamental en Java para mostrar texto en consola:

  • System.out.print(): Muestra el contenido y deja el cursor justo al final de lo impreso. No añade un salto de línea.
  • System.out.println(): Muestra el contenido y añade automáticamente un salto de línea al final, moviendo el cursor al inicio del siguiente renglón. 

        // print
        System.out.print("Hola");
        System.out.print("Mundo");
        // Salida: HolaMundo
        
        // println
        System.out.println("Hola");
        System.out.println("Mundo");
        // Salida:
        // Hola
        // Mundo

5. Salida formateada (System.out.printf)

Para presentaciones de datos más limpias (como reportes o precios), System.out.println puede quedarse corto. printf nos permite definir una "plantilla" de salida: 


        double precio = 19.995;
        // %.2f formatea a 2 decimales
        System.out.printf("El precio total es: $%.2f %n", precio); 
        // Salida: El precio total es: $20.00

6. Lectura de archivos con BufferedReader

Cuando el volumen de datos es grande, no podemos depender de la consola. Java utiliza la clase BufferedReader junto con FileReader para leer archivos de forma eficiente. Este método es más rápido que Scanner para archivos de texto extensos. 


        import java.io.BufferedReader;
        import java.io.FileReader;
        import java.io.IOException;

        public class LectorArchivo {
            public static void main(String[] args) {
                // Usamos el bloque try-with-resources para cerrar el archivo automáticamente
                try (BufferedReader br = new BufferedReader(new FileReader("datos.txt"))) {
                    String linea;
                    while ((linea = br.readLine()) != null) {
                        System.out.println(linea);
                    }
                } catch (IOException e) {
                    System.err.println("Error al leer el archivo: " + e.getMessage());
                }
            }
        }

💡 Nota: Fíjate en el bloque try (...): esto es una característica de Java llamada try-with-resources. Asegura que el archivo se cierre correctamente al terminar, evitando fugas de memoria o bloqueos en el sistema operativo, incluso si ocurre un error durante la lectura. ¡Es la forma segura de programar en Java!
