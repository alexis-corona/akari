Operadores y Control de Flujo

Dominando la toma de decisiones y los ciclos de ejecución en Java.

El control de flujo es lo que permite que un programa tome decisiones y repita tareas de forma inteligente. En Java, esto es posible gracias a una serie de estructuras que dictan la ruta de ejecución del código basándose en condiciones lógicas.

1. Operadores

Antes de controlar el flujo, debemos poder comparar valores. Java nos permite evaluar condiciones usando:

Operadores relacionales:

  • == (igual a)
  • != (distinto de)
  • > (mayor que)
  • < (menor que)
  • >= (mayor o igual que)
  • <= (menor o igual que)

Operadores lógicos:

  • && (and lógico)
  • || (or lógico)
  • ! (not lógico)

2. Estructura if/else

El if/else es la estructura de decisión básica. Permite ejecutar un bloque de código si una condición es verdadera, y opcionalmente otro si es falsa. Cuando es necesario evaluar múltiples condiciones excluyentes, utilizamos else if. 


        // if/else básico
        int edad = 21;
        if (edad >= 18) {
            System.out.println("Eres mayor de edad.");
        } else {
            System.out.println("Eres menor de edad.");
        }
        // Sailida: Eres mayor de edad.
        

        // if/else con múltiples condiciones excluyentes
        int calificacion = 45;
        if (calificacion >= 90) {
            System.out.println("Excelente");
        } else if (calificacion >= 80) {
            System.out.println("Muy bien");
        } else if (calificacion >= 70) {
            System.out.println("Bien");
        } else if (calificacion >= 60) {
            System.out.println("Suficiente");
        } else {
            System.out.println("Reprobado");
        }
        // Salida: Reprobado

3. Estructura switch

El switch es ideal cuando necesitas evaluar una variable contra múltiples valores posibles, haciendo el código mucho más limpio que múltiples if else anidados. 


        int dia = 3;
        switch (dia) {
            case 1: System.out.println("Lunes"); break;
            case 2: System.out.println("Martes"); break;
            case 3: System.out.println("Miércoles"); break;
            case 4: System.out.println("Jueves"); break;
            case 5: System.out.println("Viernes"); break;
            case 6: System.out.println("Sábado"); break;
            case 7: System.out.println("Domingo"); break;
            default: System.out.println("Día no identificado");
        }
        // Salida: Miércoles

4. Ciclo for

El for se utiliza cuando conoces de antemano el número exacto de iteraciones que deseas realizar. 


        for (int i = 0; i < 3; i++) {
            System.out.println("Iteración número: " + i);
        }
        // Salida:
        // Iteración número: 0
        // Iteración número: 1
        // Iteración número: 2

5. Ciclo while

El while repite un bloque de código mientras una condición permanezca verdadera. Se evalúa antes de cada iteración. 


        int contador = 0;
        while (contador <= 5) {
            System.out.println("Contador: " + contador);
            contador++;
        }
        // Salida:
        // Contador: 0
        // Contador: 1
        // Contador: 2
        // Contador: 3
        // Contador: 4
        // Contador: 5

6. Ciclo do-while

A diferencia del while, el do-while garantiza que el bloque de código se ejecute al menos una vez, ya que la condición se evalúa al final. 


        int x = 0;
        do {
            System.out.println("Esto se ejecuta aunque la condición sea falsa después.");
            x++; // aquí x = 1
        } while (x < 1);  // como x = 1, la condición ya no es verdadera
        // Salida: Esto se ejecuta aunque la condición sea falsa después.

💡 Nota: Elegir la estructura correcta es fundamental. Mientras que un for es excelente para recorrer colecciones de datos, un while es más apropiado cuando esperas un evento externo o una condición dinámica. ¡Domina estas herramientas para escribir algoritmos eficientes!
